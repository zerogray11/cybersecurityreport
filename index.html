🔍 2. Detailed Vulnerability Analysis

🚨 Vulnerability #1: UnrealIRCd Backdoor

Critical Remote Code Execution

CRITICAL
Target System
192.168.56.103
Service
UnrealIRCd on port 6667
Exploit Module
exploit/unix/irc/unreal_ircd_3281_backdoor
Impact
Remote root shell access

Summary

A backdoored version of UnrealIRCd was discovered running on port 6667 of the Metasploitable2 machine. Using Metasploit's unreal_ircd_3281_backdoor module, remote root shell access was obtained. This vulnerability stems from a maliciously modified source distribution of UnrealIRCd that allows attackers to execute arbitrary commands without authentication.

🔧 Exploitation Steps

  1. Service Discovery: Conducted Nmap scan against target system
  2. Vulnerability Identification: Identified UnrealIRCd 3.2.8.1 on port 6667
  3. Exploit Selection: Used Metasploit's search command to find appropriate exploit
  4. Configuration: Set RHOST, LHOST, and payload parameters
  5. Execution: Successfully obtained root shell access

Figure: Metasploit search command showing UnrealIRCd backdoor exploit

Figure: RHOST & LHOST setup

Figure: Payload selection and final config

Figure: Root shell access confirmed via whoami and uname -a

🛠️ Remediation

  • Immediately upgrade UnrealIRCd to the latest secure version
  • Remove the backdoored installation completely
  • Configure firewall to block unused ports
  • Implement network segmentation
  • Regular security audits of all installed software

⚠️ Vulnerability #2: Samba Usermap Script

Command Injection Vulnerability

HIGH
Target System
192.168.56.103
Service
Samba on ports 139, 445
Exploit Module
exploit/multi/samba/usermap_script
Impact
Remote root shell access

Summary

An outdated and misconfigured Samba service was discovered on ports 139 and 445. The usermap_script vulnerability was exploited using Metasploit, allowing remote code execution via a specially crafted username parameter. This vulnerability demonstrates how insecure configurations combined with legacy services can lead to complete system compromise.

🔧 Exploitation Steps

  1. Port Enumeration: Identified Samba services on ports 139 and 445
  2. Vulnerability Research: Found usermap_script exploit in Metasploit
  3. Exploit Configuration: Set target parameters and payload
  4. Command Injection: Successfully executed arbitrary commands
  5. Shell Access: Obtained root shell on target system

🔶 Vulnerability #3: Tomcat Manager Brute-force Attack Surface

Exposed Management Interface

MEDIUM
Target System
192.168.56.104
Service
Apache Tomcat 9.0.7 on port 8080
Exploit Module
auxiliary/scanner/http/tomcat_mgr_login
Impact
Interface exposed to brute-force attacks

Summary

Apache Tomcat Manager was exposed on port 8080 of the Basic Pentesting 2 machine. A brute-force attack was conducted using Metasploit's tomcat_mgr_login module and common username/password combinations. While no valid credentials were cracked, the interface accepted all login attempts with no throttling or rate-limiting. Manual testing of default credentials (admin:admin) was also performed. This confirmed that the endpoint was publicly accessible and vulnerable to brute-force or dictionary attacks, highlighting poor password policies and weak security configurations.

🔧 Exploitation Steps

  1. Port Scanning: Conducted full port scan with version detection
  2. Service Identification: Identified Apache Tomcat 9.0.7 on port 8080
  3. Wordlist Preparation: Installed wordlists package and extracted rockyou.txt
  4. Brute-force Attack: Used Metasploit's tomcat_mgr_login module
  5. Manual Testing: Tested default credentials via web browser

Figure: Full Nmap scan revealing Apache Tomcat on port 8080 of 192.168.56.104

Figure: Installing and extracting rockyou.txt for dictionary attacks

Figure: Metasploit module configured with RHOSTS, RPORT, and rockyou.txt for brute-force attack

Figure: Brute-force attempts showing no lockout or rate limiting in Apache Tomcat Manager

Figure: Apache Tomcat Manager login interface accessed via browser at port 8080

🛠️ Remediation

  • Disable default Tomcat manager interface if not required
  • Enforce strong password policies for all accounts
  • Implement rate-limiting for authentication attempts
  • Configure firewall to block unauthenticated access
  • Enable account lockout mechanisms after failed attempts
  • Monitor and log all authentication attempts

🎯 4. Conclusions and Recommendations

Assessment Results

This assessment successfully identified and exploited multiple vulnerabilities, including two that resulted in remote root access and one critical brute-force attack surface in a secondary VM. The findings demonstrate significant security weaknesses that could be exploited by malicious actors to gain unauthorized access to sensitive systems and data.

Risk Summary

Critical Risk: Backdoored UnrealIRCd service providing immediate root access
High Risk: Samba command injection vulnerability enabling system compromise
Medium Risk: Exposed Tomcat Manager interface vulnerable to brute-force attacks

🚀 Priority Recommendations

  • Immediate Actions: Patch or remove all identified vulnerable services
  • Network Security: Implement proper firewall rules and network segmentation
  • Access Control: Enforce strong authentication and authorization mechanisms
  • Monitoring: Deploy comprehensive logging and intrusion detection systems
  • Regular Audits: Establish periodic security assessments and vulnerability scanning
  • Patch Management: Implement systematic patching procedures for all systems

Final Notes

These results highlight the importance of regular patching, secure configuration, and proper service hardening to prevent privilege escalation or data exposure. Organizations should prioritize addressing critical and high-severity vulnerabilities immediately, followed by implementing comprehensive security controls to prevent similar issues in the future.
